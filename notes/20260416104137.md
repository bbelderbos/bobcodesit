# Use Protocol to decouple your LLM provider

Hardcoding `openai.chat.completions.create(...)` throughout your app couples you to one vendor. Define a `Protocol` instead. Your service layer depends on the interface, and the concrete provider is injected at startup. Swap providers (OpenAI, Groq, local model) or inject a test fake without changing anything else.

```python
from decimal import Decimal
from typing import Protocol, Sequence


class Assistant(Protocol):
    def completion(self, messages: list[dict[str, str]]) -> ExpenseCategorizationResponse: ...
    def calculate_cost(self, prompt_tokens: int, completion_tokens: int) -> Decimal: ...
    def get_available_models(self) -> Sequence[str]: ...


# OpenAIAssistant and GroqAssistant satisfy this without inheriting from it
class OpenAIAssistant:
    def completion(self, messages: list[dict[str, str]]) -> ExpenseCategorizationResponse:
        ...

    def calculate_cost(self, prompt_tokens: int, completion_tokens: int) -> Decimal:
        ...

    def get_available_models(self) -> Sequence[str]:
        ...


# Service accepts the Protocol — provider is injected, not hardcoded
@dataclass
class ClassificationService:
    assistant: Assistant  # any conforming impl works here

    def classify(self, description: str) -> ClassificationResult:
        ...
```

This is structural typing: the type checker enforces the contract, no inheritance needed. Concrete classes just need to implement the right methods; duck typing with safety.
