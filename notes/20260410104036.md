# StrEnum and auto()

If you're still using standard `Enum` for states, categories, or API keys, you're missing out.

In Python >= 3.11, `StrEnum` combined with `auto()` is a cleaner way.

Wins:
- Direct comparison: a `StrEnum` member *is* a string. No more `if status.value == "active":`.
- Auto-naming: `auto()` automatically assigns the lowercase version of the variable name as the value.
- Serialization: since they inherit from `str`, they work wellwith `json.dumps()` and database drivers without custom encoders.

Comparison code below 👇

```python
from enum import Enum

class Status(Enum):
    PENDING = "pending"

# This fails! Standard Enums aren't strings.
if Status.PENDING == "pending": 
    pass 

# You're forced to do this:
print(Status.PENDING.value) 

# more modern
from enum import StrEnum, auto

class Status(StrEnum):
    PENDING = auto()  # Value is automatically "pending"
    ACTIVE = auto()   # Value is automatically "active"

# This works perfectly!
if Status.PENDING == "pending":
    print("It's a match!")

# Works in f-strings and APIs without .value
print(f"Current status: {Status.ACTIVE}") 
```

Useful for state machines, API clients and configs. Have you used this yet?
