# LBYL vs EAFP: Two Pythonic Styles

Python offers two approaches for handling uncertain operations: check first (LBYL) or try and catch (EAFP).

```python
config = {"debug": True, "timeout": 30}

# LBYL: Look Before You Leap
# Check condition, then act. Explicit, but can race.
if "timeout" in config:
    timeout = config["timeout"]
else:
    timeout = 60

# EAFP: Easier to Ask Forgiveness than Permission
# Try the operation, handle failure. More Pythonic.
try:
    timeout = config["timeout"]
except KeyError:
    timeout = 60
```

EAFP is generally preferred in Python because:
- It's atomic (no race between check and access)
- It's often faster when the key/attribute usually exists
- It reads as "do the thing" rather than "check, then do"

That said, LBYL can be clearer when the check itself is informative or when exceptions are expensive. Use EAFP when failure is rare; use LBYL when you need the check's result or when exceptions have high overhead.
