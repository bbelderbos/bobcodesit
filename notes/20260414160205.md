# asyncio.gather() fires coroutines concurrently — sequential await does not

Using `await` in a loop runs each coroutine one after the other. `asyncio.gather()` starts them all at once and waits for the slowest, not the sum.

```python
import asyncio
import time

# Simulate a network request that takes `delay` seconds
async def fake_fetch(name: str, delay: float) -> str:
    await asyncio.sleep(delay)
    return f"{name} done"

async def sequential():
    for name, delay in [("A", 1.0), ("B", 1.0), ("C", 1.0)]:
        await fake_fetch(name, delay)

async def concurrent():
    await asyncio.gather(
        fake_fetch("A", 1.0),
        fake_fetch("B", 1.0),
        fake_fetch("C", 1.0),
    )

for label, fn in [("sequential", sequential), ("concurrent", concurrent)]:
    t = time.perf_counter()
    asyncio.run(fn())
    print(f"{label}: {time.perf_counter() - t:.2f}s")

# sequential: 3.00s
# concurrent: 1.00s
```

Works because `asyncio.sleep()` (like real network I/O) yields control back to the event loop — other coroutines run during the wait. Doesn't help CPU-bound work.
