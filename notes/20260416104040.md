# Use create_autospec when mocking your own Protocols

`MagicMock()` accepts any attribute access silently. If you rename a method in your Protocol but forget to update the mock, the test still passes, a false green. `create_autospec` enforces the real interface at call time, so signature drift fails loudly.

```python
from unittest.mock import create_autospec

# Bad: mock_assistant accepts any attribute, even typos
mock_assistant = MagicMock()

# Good: mock enforces the Assistant Protocol's actual interface
mock_assistant = create_autospec(Assistant)
mock_assistant.completion.return_value = ExpenseCategorizationResponse(...)

service = ClassificationService(assistant=mock_assistant)
result = service.classify("Coffee $5.50")

mock_assistant.completion.assert_called_once()
```

Use it whenever you're mocking one of your own Protocols or classes. Leave `MagicMock` for external library objects (e.g. Telegram `Update`, `requests.Response`) where you're not trying to enforce your own contracts.
