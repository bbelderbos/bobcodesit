# Module-level objects are global mutable state — use app.state or fixtures instead

Objects created at import time are shared across every request and every test. Tests that "clean up" a module-level database engine aren't isolated — they share the same connection pool and auto-increment counters.

```python
# Trap: engine created once at import, shared everywhere
engine = create_engine("sqlite:///app.db")  # module-level

# Fix 1: per-test in-memory engine via pytest fixture
@pytest.fixture
def engine():
    eng = create_engine("sqlite://")  # in-memory, isolated
    SQLModel.metadata.create_all(eng)
    yield eng
    eng.dispose()

# Fix 2: FastAPI — store shared state on app, not at module level
from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app: FastAPI):
    app.state.data_source = build_data_source()
    yield

app = FastAPI(lifespan=lifespan)

def get_data_source(request: Request):
    return request.app.state.data_source  # one instance, easy to override in tests
```

Rule of thumb: if an object holds mutable state, don't create it at module scope. Control its lifetime explicitly.
