# Django ORM: `get_or_create` vs `update_or_create`

Both avoid manual existence checks, but they differ in what happens when a record *already exists*.

**`get_or_create`** — protective. If found, returns it unchanged. If not, creates it.
Use for stable reference data (tags, categories) where you just need the record to exist.

**`update_or_create`** — proactive. If found, updates it with `defaults`. If not, creates it.
Use when syncing external data that can change over time.

```python
# Syncing from an external API — title may change, so update it
anime_obj, created = Anime.objects.update_or_create(
    media_id=anime["id"],
    defaults={
        "title": anime["title"]["english"],
        "country_of_origin": anime["countryOfOrigin"],
    },
)

# Ensuring a tag exists — don't overwrite anything
genre, created = Genre.objects.get_or_create(name="Action")
```

If you used `get_or_create` for the API sync, a title change ("Attack on Titan" → "AOT: Final Season") would silently stay stale in your DB.
