# The `?` operator in Rust

"I expect this to work; if it doesn't, we're done here."

You can express this cleanly in Rust using `?`.

It allows you to maintain focus on the business logic rather than error-handling mechanics.

```rust
// Without ? - verbose error propagation
fn parse_json(input: &str) -> Result<JsonValue, JsonError> {
    let tokens = match tokenize(input) {
        Ok(t) => t,
        Err(e) => return Err(e),
    };

    let value = match parse_tokens(&tokens) {
        Ok(v) => v,
        Err(e) => return Err(e),
    };

    Ok(value)
}

// With ? - clean and focused
fn parse_json(input: &str) -> Result<JsonValue, JsonError> {
    let tokens = tokenize(input)?;
    let value = parse_tokens(&tokens)?;
    Ok(value)
}
```
