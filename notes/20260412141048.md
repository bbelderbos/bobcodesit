# Sending email with the Resend SDK in Python

I recently switched from SendGrid to Resend. 3K monthly free emails, minimal setup and a simple API.

Install: `uv add resend`

To send an email, set your API key and call `resend.Emails.send()` with the appropriate parameters:

```python
import resend
from decouple import config

resend.api_key = config("RESEND_API_KEY")

params = {
    "from": "Me <noreply@mydomain.com>",
    "to": ["user@example.com"],
    "subject": "Welcome!",
    "html": "<strong>Hello!</strong>",
    "text": "Hello! (Plain text version)", # let email client choose html or text
    # could also add different reply to email:
    # "reply_to": ["support@mydomain.com"],
}

# catch-all, Resend SDK does provide granular exceptions
# if you want to handle specific cases
try:
    response = resend.Emails.send(params)
except Exception:
    logging.exception("ERROR sending email")
```

Note: make sure mydomain.com is verified in your Resend dashboard, or the API will return a 403 Forbidden error.

Key points:
- Set `resend.api_key` before calling `send()`
- Provide `"html"`, `"text"`, or both. Using both ensures the best compatibility across all mail clients.
- `"to"` and `"reply_to"` take lists, even for a single address
- `"from"` accepts `"Name <email>"` format directly
- Wrap in try/except, the SDK raises on non-2xx responses (unlike SendGrid which returns a status code you check manually)
