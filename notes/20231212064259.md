# The power of context managers

Context Managers, a must-use 🐍 Python feature 😍

Here's why they're awesome:

• Resource Management: automate setup/teardown for cleaner, leak-proof code.

• Error Handling: manage exceptions elegantly, ensuring resource release.

• Code Clarity: `with` statements define clear resource usage scope, increasing readability.

Well known for file handling, another example where context managers shine is managing database connections (example below).

There are two ways to define them:

• Class-based: adding `__enter__` + `__exit__ dunder methods.

• Function-based: decorating a function with `@contextmanager` (contextlib module).

Here is an example of both:

```
import sqlite3
from contextlib import contextmanager
from typing import Generator, Optional, Type, Any

# 1. class-based


class DatabaseConnection:
    def __init__(self, db_file: str) -> None:
        self.db_file = db_file
        self.connection: Optional[sqlite3.Connection] = None

    def __enter__(self) -> sqlite3.Connection:
        self.connection = sqlite3.connect(self.db_file)
        return self.connection

    def __exit__(self, exc_type, exc_val, exc_tb) -> None:
        if self.connection is not None:
            self.connection.close()


# test
db_name = ":memory:"  # Use in-memory DB for quick testing
with DatabaseConnection(db_name) as conn:
    print(f"Class Test: Connection open? {bool(conn)}")
    # Verify we can execute commands
    conn.execute("CREATE TABLE test (id INT)")

# The __exit__ method is called here.
# We can't easily check conn.closed directly on sqlite3 objects,
# but attempting a query outside the block will confirm the state.
try:
    conn.execute("SELECT 1")
except sqlite3.ProgrammingError:
    print("Class Test: Connection successfully closed.")


# 2. function-based


@contextmanager
def database_connection(db_file: str) -> Generator[sqlite3.Connection, None, None]:
    connection = sqlite3.connect(db_file)
    try:
        yield connection
    finally:
        # This block runs even if an exception occurs in the 'with' body
        connection.close()


# test
with database_connection(":memory:") as conn:
    print(f"Function Test: Connection open? {bool(conn)}")
    conn.execute("CREATE TABLE test (id INT)")

try:
    conn.execute("SELECT 1")
except sqlite3.ProgrammingError:
    print("Function Test: Connection successfully closed.")

```
