# Exhaustive match in Python with StrEnum + assert_never

Python's `match` is nice, but by default it does not warn you when you forget a case. Combine `StrEnum` for typed values, `match` for dispatch, and `typing.assert_never` in the catch-all, and a type checker will flag any unhandled variant before runtime.

Add a new enum member tomorrow, and the dispatcher fails type-checking until you handle it. Closest thing Python has to Rust's exhaustiveness.

```python
from enum import StrEnum
from typing import assert_never

class Provider(StrEnum):
    ANTHROPIC = "anthropic"
    OPENAI = "openai"

def make_ranker(provider: Provider) -> str:
    match provider:
        case Provider.ANTHROPIC:
            return "anthropic-client"
        case Provider.OPENAI:
            return "openai-client"
        case _:
            assert_never(provider)
```

Add `GROK = "grok"` to the enum and ty complains:

```bash
error[type-assertion-failure]: Argument does not have asserted type `Never`
  --> a.py:18:13
   |
18 |             assert_never(provider)
   |             ^^^^^^^^^^^^^--------^
   |                          |
   |                          Inferred type of argument is `Literal[Provider.GROK]`
   |
info: `Never` and `Literal[Provider.GROK]` are not equivalent types
```

This basically says: "You told me that this _ case should be unreachable (Never), but I found a scenario where it is reachable: when provider is Provider.GROK. Therefore, your code is not exhaustive."

Adding:

```python
        case Provider.GROK:
            return "grok-client"
```

to `make_ranker` makes the type checker happy again.
