# pathlib / operator overloading

Ever wondered how pathlib.Path() overloads the "/" in an operation like:

`Path(dirname) / filename`

The power of Python's data model: the `Path` class implements the `__truediv__` dunder method.

```
class SmartPath:
    def __init__(self, path):
        self.path = path

    # This is how Path handles the / operator
    def __truediv__(self, other):
        # We join the current path with the 'other' string
        new_path = f"{self.path}/{other}"
        return self.__class__(new_path)

    def __repr__(self):
        return f"{self.__class__.__name__}({self.path!r})"


a = SmartPath("/home") / "user" / "documents"
print(a)  # Output: SmartPath('/home/user/documents')
```
