# Unix piping with Claude

TIL more powerful Unix piping, using Claude Code's `-p` flag (print mode), which accepts piped input.

For example, when tests fail, you can get a fast root-cause read without copy-pasting tracebacks into a chat window.

```bash
uv run pytest tests/unit/test_week3.py::TestDBExpenseRepo::test_db_expense_repo_search_by_dates -q
```

Side note: if you intended the upper bound to be exclusive (a common convention to make day-bucket queries cleaner), the test and the in-memory implementation also need to change to match — but right now the contract across the abstract class, in-memory impl, and tests is inclusive on both ends, so the DB impl should follow.
```

The `2>&1` matters here, because pytest writes some output to stderr, so make sure Claude sees the complete picture.

What makes this powerful is context: Claude finds the bug (here staged for this tip) + warns about related issues / risks. 

Another cool example is combining it with git and other unix commands, here summarizing a big commit + outputting json :

```bash
$ git show 9db8301 | claude -p "Summarize this diff in one sentence. Return JSON: {\"summary\": \"...\"}" --output-format json | jq -r '.result' | sed '/^```/d' | jq -r '.summary'

Redesigns the homepage with a cleaner, professional layout (new hero, stats row, problem/approach, feature cards, refreshed testimonials, and CTA sections) and moves the 8-persona content to a new /who-is-this-for/ page.
```
