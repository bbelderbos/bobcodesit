# Power of type hints in modern Python libraries and frameworks

Modern Python frameworks read type annotations at runtime. Define your model once and multiple layers of the app derive behavior from the same source of truth.

```python
from enum import StrEnum, auto
from sqlmodel import SQLModel, Field

class Language(StrEnum):
    PYTHON = auto()
    RUST = auto()
    JS = auto()

class Snippet(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    title: str
    code: str
    language: Language = Language.PYTHON
    is_favorite: bool = False
```

What each annotation does across the stack:
- **SQLModel** → generates PostgreSQL columns (`TEXT`, `BOOLEAN`, `VARCHAR` enum)
- **FastAPI** → validates incoming JSON, returns 422 on bad input, no extra code
- **Typer** → `Language` becomes a validated CLI choice; `bool` becomes `--flag/--no-flag`
- **FastAPI /docs** → OpenAPI schema always matches the model, auto-generated

Pydantic (and SQLModel) is the runtime counterpart to static type checking: types tell the checker what *should* be true, Pydantic enforces what *is* true at the boundary.
