# match/case as a lightweight finite state machine

Python 3.10+ `match/case` can replace a long if/elif/elif chain for simple state dispatchers. Tuple matching on `(current_state, action)` reads almost like a transition table:

```python
def next_state(current: str, action: str) -> str:
    match (current, action):
        case ("draft", "open"):     return "open"
        case ("open", "approve"):   return "approved"
        case ("open", "close"):     return "closed"
        case ("approved", "merge"): return "merged"
        case ("closed", "reopen"):  return "open"
        case _:
            raise ValueError(f"Invalid transition: {action!r} from {current!r}")

assert next_state("draft", "open") == "open"
assert next_state("approved", "merge") == "merged"
```

Clean and readable for a limited amount of states. Once you need guards ("can't merge without approvals"), hooks (fire a callback on transition), or history tracking, so a full FSM, reach for a class (or lib), see article: https://belderbos.dev/blog/build-finite-state-machine-python/
