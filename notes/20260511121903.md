# s/float/Decimal/g for money

Floats can't represent decimal fractions exactly. Lucky for money Python has the `Decimal` type, which is designed for financial applications. Why it's better than floats:

```python
>>> 10.50 + 0.10
10.600000000000001

>>> from decimal import Decimal
>>> Decimal("10.50") + Decimal("0.10")
Decimal('10.60')
```

A few rules to follow:

- Construct from strings, not floats. `Decimal("0.1")` is exact; `Decimal(0.1)` inherits the same binary error you were trying to escape.

- Annotate your models (Pydantic, SQLModel, dataclasses) as `Decimal`, and map to `NUMERIC`/`DECIMAL` columns in the DB, not `FLOAT`.

- After division, use `.quantize(Decimal("0.01"))` to define the rounding precision. 

Avoid rounding errors and unexpected pennies in your financial calculations, use `Decimal` for money.
